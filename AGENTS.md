# Repository Contribution Guidelines

## Workflow
- Use `pnpm` for all dependency, lint, and build commands.
- Run `pnpm lint` and `pnpm build` before committing changes.
- Keep commit messages and any in-code comments in English.

## Code Quality
- Prioritize scalability, readability, and long-term maintainability in all changes.
- Align new code with existing project structure and conventions.
- Favor clear naming, small functions, and explicit typing when possible.

## Documentation and Communication
- Update relevant documentation when behavior or configuration changes.
- When adding PR descriptions, include a concise summary of changes and validation steps.
